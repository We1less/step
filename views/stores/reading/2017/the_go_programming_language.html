《Go程序设计语言》

zhgxun1989 2017年07月02日

历时一个月，零碎的腾出每天晚上或者早上的1小时，《The Go Programming Language》(go程序设计语言)这本书，除最后三章(测试，反射和低阶程序设计(这三章暂时不看))外，算是慢慢对着网上的中文教程(http://books.studygolang.com/gopl-zh/)厚颜无耻的把英文版给看完了。

较PHP来说，Go是一门年轻的编程语言，没有那么多的库和语法糖开箱即用。在此之前，我曾花费将近一年零散的时间来学习Python，最后因为MySQL库连接无比费劲和版本之间的差异，好多书买回来还没来得及看，我就不再尝试Python了。但是Go和它们比起来，显得简洁但强大的多。是的，你可能真的不敢相信，它的语法其实很少，语言层面没有那么多需要学习的所谓知识。

“Go项目不仅仅包含语言本身、它的工具和标准库，还有决不能忽视的一点，就是它保持极端简单性的行为文化...相对来说，它的语言特性不多，而且不太会增加新特性了。比如说，它没有隐式数值类型强制转换，没有构造或析构函数，没有运算符重载，没有形参默认值，没有继承，没有泛型(据说go1.9版本增加泛型支持了，该版本还未发布)，没有异常，没有宏，没有函数注记，没有线程局部存储。”

当然，也没有三元运算符，只有一个for循环...

毕竟是Web开发出身，还是PHP这种世界上最好的语言，会关注更多Web开发方面的事。

“Go语言目前没有一个权威的web框架，就像Ruby语言有Rails和python有Django。这并不是说这样的框架不存在，而是Go语言标准库中的构建模块就已经非常灵活以至于这些框架都是不必要的。此外，尽管在一个项目早期使用框架是非常方便的，但是它们带来额外的复杂度会使长期的维护更加困难。”

看了下beego的文档，尝试写了一个简单的博客，还在segmentfault上写了一篇简单的小教程，用beego写一个友情链接。虽然套路都一样，但还是有很多区别，跟解释型语言比起来，编译型语言就没那么便捷了。每一种语言都有它擅长的领域，工具本身就应该用它来做它擅长的事情。
